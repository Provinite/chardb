Image Moderation Queue

A walkthrough of the content moderation workflow for CharDB

0

Overview

The Image Moderation Queue ensures all uploaded images are reviewed before becoming publicly visible. This helps maintain community standards and prevents inappropriate content from appearing on the platform.

What Users See While Pending

When an image is uploaded, it enters the moderation queue with a PENDING status. During this time, all users—including the uploader—see a placeholder image instead of the actual content.

The placeholder clearly indicates that the image is awaiting moderation. This ensures that:

  • Unapproved content is never displayed publicly
  • Users know their upload was received and is being processed
  • The media entry still appears in galleries and character pages (with the placeholder)
  • Only moderators can view the actual image content, and only through the moderation queue
Note: Media metadata (title, description, character association) is still visible. Only the actual image content is replaced with the placeholder.
A media page showing the "Image Pending Moderation" placeholder while awaiting review
2

Accessing the Moderation Queue

Community moderators and administrators can access the Image Moderation Queue from the Administration section in the community sidebar. Navigate to Moderation to view all pending images.

The queue displays a count badge showing how many images are awaiting review. In this example, there is 1 pending image in the queue.

The Content Moderation page showing the Image Moderation Queue with pending count badge
3

Reviewing Pending Images

Each pending image is displayed as a card containing all the information needed to make a moderation decision:

  • Image Preview: A thumbnail of the uploaded image (click to view full size)
  • Uploader: Link to the user who uploaded the image
  • Character: The character this image is associated with
  • Community: Which community the image belongs to
  • Timestamp: When the image was uploaded
Tip: Click on the image thumbnail to open a full-size view for detailed inspection before making your decision.

At the bottom of each card are two action buttons:

  • Approve - Makes the image publicly visible
  • Reject - Hides the image and notifies the uploader
A moderation card showing image metadata and action buttons
4

Approving an Image

If the image meets community guidelines, click the green Approve button. This will:

  • Set the image's moderation status to APPROVED
  • Make the image visible to all community members
  • Send an email notification to the uploader confirming approval
  • Remove the image from the moderation queue
Note: Approval is immediate. A success toast will appear confirming the action, and the queue will automatically refresh.
5

Rejecting an Image

If an image violates community guidelines, click the red Reject button to open the rejection modal. You must select a reason for the rejection:

  • Terms of Service Violation: Content violates community guidelines or terms of service
  • NSFW Not Tagged: Adult content that was not properly marked as NSFW
  • Spam / Low Quality: Spam, duplicate content, or extremely low quality images
  • Copyright Issue: Potential copyright infringement or unauthorized use
  • Other: Other reason not listed above (requires explanation)

You can also provide Additional Details to give the uploader more context about why their image was rejected. This message will be included in the notification email.

Important: If you select "Other" as the reason, you must provide additional details explaining the rejection. The "Reject Image" button remains disabled until all required fields are completed.
The rejection modal with predefined reasons and optional details field
6

After Moderation

After processing an image, the queue automatically refreshes. When all pending images have been reviewed, you'll see the "All Caught Up!" message indicating the queue is empty.

All moderation actions are logged for audit purposes, including:

  • Which moderator took the action
  • The action taken (approve/reject)
  • The rejection reason (if applicable)
  • Any additional details provided
  • Timestamp of the action
Email Notifications: Uploaders receive an email for both approvals and rejections. Rejection emails include the reason and any additional details you provided, helping users understand how to improve future uploads.
