Character Deletion & Species Removal

A walkthrough of the community admin tools for managing characters

0

Overview

Community admins and moderators now have two new character management tools: Delete Character and Remove from Species. These actions are available directly on the character page and, for efficiency, on every card in the Trait Review Queue.

🗑️

Delete Character

Soft-deletes the character. Reversible by a site admin.

↩️

Remove from Species

Detaches the character from its species, preserving trait data as custom fields.

🛡️

Permission-gated

Requires the canDeleteCharacter role permission or global admin.

📋

Queue integration

Both actions are available inline on Trait Review cards.

1

Action buttons on the character page

When viewing a character that belongs to your community, three action buttons appear in the top-right of the character card — provided you have the canDeleteCharacter permission (or are a global site admin):

  • Edit Character — opens the character edit form
  • Remove from Species — detaches the character from its species (only shown when the character has a species assigned)
  • Delete Character — soft-deletes the character
Note: The buttons only appear while the character is viewed in a community context (i.e., the character belongs to a species within your community). A character that has already been removed from its species will not show community action buttons.
Character page viewed as a community admin — the three action buttons appear alongside the character owner info
2

Deleting a character

Clicking Delete Character shows a confirmation dialog describing what will happen:

  • The character is soft-deleted — it is hidden from all public views but the data is retained in the database.
  • Any pending trait reviews for the character are cancelled.
  • After confirmation you are redirected to the character browse page.
Important: Soft-deletion is intentionally not self-service reversible. Recovery requires a site admin to manually clear the deletedAt timestamp. Use this action for characters that violate community rules or are otherwise no longer appropriate.

A separate Purge mutation is available to site admins only via the GraphQL API for hard (permanent) deletion when needed.

3

Removing a character from its species

Clicking Remove from Species shows a confirmation dialog before proceeding. The operation:

  • Clears the character's species, variant, registry ID, and trait values.
  • Preserves the trait data — all existing trait values are flattened into the character's custom fields (e.g., Eye Color → Amber becomes a custom field entry) so no information is lost.
  • Cancels any pending trait reviews for that character.
  • The character remains visible on its owner's profile and in global browse — it simply no longer appears under the species in the community.
Tip: Use this action when a character was registered to a species in error, or when the owner requests removal from the species registry without losing their character entirely.
After removal: the character's Eye Color trait value ("Amber") is preserved as a custom field — no data is lost
4

Actions in the Trait Review Queue

Both actions are also available directly on each card in the Trait Review Queue. This lets moderators handle problematic submissions without leaving the queue — approve legitimate traits, remove a character from the species if the traits are incorrect, or delete the character if it violates community rules.

  • Approve — confirms the proposed trait values
  • Remove from Species — kicks the character from the species, flattening its traits to custom fields and removing the review entry from the queue
  • Delete — soft-deletes the character, which also cancels the pending review and removes it from the queue
Tip: The Reject button (for sending the character's traits back to the previous approved state) is still available for cases where you want to roll back values without removing the character from the species.
Trait Review card with the new Remove from Species and Delete action buttons alongside the existing Approve button
5

Setting up the permission

Both actions are gated by the canDeleteCharacter permission on a community role. To grant it:

  • Go to Community Settings → Roles and open the role you want to configure (e.g., Admin or Moderator).
  • Enable the Can Delete Character toggle.
  • Save the role.

Members with that role will immediately see the action buttons on character pages and in the trait review queue.

Default roles: The Admin role created automatically when a new community is started now includes canDeleteCharacter by default. Moderator and Member roles do not.
