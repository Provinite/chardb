Trait Clarifiers

A walkthrough of free-text clarifiers on character trait values

0

Overview

Clarifiers let you attach an optional free-text note to any character trait value. The clarifier is rendered in parentheses after the value, so a single trait can describe many distinct things without exploding the trait list.

For example, instead of creating a separate trait for every body modification, you can use one multi-value Body Mod trait and clarify each instance:

Common Body Mod (extra horns)
Common Body Mod (stripes on tail)
Rare Body Mod (iridescent wings)

1

Enable clarifiers on a trait

Open the species' Trait Builder, then either create a new trait or edit an existing one. Toggle Allow clarifier text on. The setting lives on the trait itself, so it applies to every variant that uses the trait.

Tip: Pair clarifiers with Allow multiple values to model grouping traits (e.g. one Body Mod trait with multiple clarified instances). They work independently too — a single-value trait can still carry a clarifier.
Trait builder Edit modal with both "Allow multiple values" and "Allow clarifier text" enabled
2

Fill in clarifiers on a character

When editing a character, traits with clarifiers enabled show a second text input labelled Clarifier (optional) next to the value input.

  • Single-value trait: type the value, then type the clarifier. Both fields update the trait independently.
  • Multi-value trait: type the value and (optional) clarifier, click Add. The entry is added as a chip with the format Value (clarifier). Repeat to add more, including duplicates of the same value with different clarifiers.
Note: The clarifier is optional. Leaving it blank stores only the bare value. The maximum clarifier length is 200 characters.
Multi-value Body Mod trait with two clarified entries and the value + clarifier inputs ready for the next addition
3

How clarifiers display

On the character profile, clarified values are rendered with the clarifier in parentheses immediately after the value. Multi-value traits show each instance as its own chip; single-value traits show the value inline.

The character profile rendering "Common Body Mod (extra horns)" and "Common Body Mod (long tail)" as separate chips
4

Clarifier changes in trait review

The Trait Review Queue treats the (value, clarifier) pair as the diff key. Changing only the clarifier — same value, different note — registers as a change, just like swapping the value itself.

  • Removed entries are shown with strikethrough text
  • Added entries are highlighted (green border / chip)
  • Unchanged entries appear normally for context
Tip: A clarifier-only edit is intentionally visible in moderation. Reviewers can spot, for example, a renamed mod ("extra horns" → "spiral horns") without having to inspect raw data.
Review diff: "Common Body Mod (long tail)" removed; "(stripes on tail)" and "Rare Body Mod (iridescent wings)" added
5

Rules & validation

  • Clarifier input is only shown for traits where the species manager has enabled Allow clarifier text. The backend rejects clarifiers submitted on traits that did not opt in.
  • A clarifier requires a value — you can't store a clarifier against an empty trait entry.
  • Maximum length is 200 characters. Surrounding whitespace is trimmed; an all-whitespace clarifier is treated as absent.
  • On enum (Selection) multi-value traits, enabling clarifiers allows the same enum option to appear multiple times with different clarifiers (e.g. two Common Body Mod entries). Without clarifiers, already selected options remain hidden from the dropdown.
